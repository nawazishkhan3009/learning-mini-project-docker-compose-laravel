# Description

A simple PHP,Laravel and Nginx Dockerized Web Application


### Architecture

App Containers:
1. Nginx Web Server
2. PHP Interpreter
3. MySQL Database

Utility Containers:
4. Composer
5. Laravel Artisan
6. npm

### Using Composer to Generate Laravel App, also run everything
``` bash
docker compose run --rm composer create-project laravel/laravel .
docker-compose up -d --build server

```

### Docker Base Image Versions
| Container    | Version |
| --------     | ------- |
| nginx        | stable-alpine    |
| PHP          | 8.4-fpm-alpine     |
| Composer     | 2.8    |
| mysql        | 8.4    |

A lot of errors arise from the incompatibility between version of Laravel, PHP and Composer.
Since we are using a separate Composer container to create-project, whichever version of composer:1.0,1.2,2.0 or 2.8 I tried, they all had PHP 8.4+ in 2026 installed in their environment. This leads to a mismatch between the Project generated by composer and the PHP version specific in our php.dockerfile.

### Database connection credentials between Laravel <> Mysql
Copy/Replace the following the test credentials to the 'src/.env' directory generated by Composer

``` bash
DB_CONNECTION=mysql
DB_HOST=mysql
DB_PORT=3306
DB_DATABASE=homestead
DB_USERNAME=homestead
DB_PASSWORD=secret
```
